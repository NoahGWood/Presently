Web Standards & The Internet

Background

  The internet has come a long way since its inception in the late 1960s. What started as a way for the US military to share information between computers has grown into a global network that is used by billions of people every day. The World Wide Web, or the web for short, is a system of interconnected documents that can be accessed via the internet. (Galileo, n.d.) The World Wide Web was invented in 1989 by Tim Berners-Lee, a nuclear physicist who was working at CERN. at the time. (Berners-Lee, 2022)

Trouble In The Early Days

  By the mid 1990's, the world wide web was taking off, and along with this novel technology came the infamous Browser Wars that took place between Netscape's Navigator Browser and Microsoft's Internet Explorer. Each company, trying to outdo one another, kept adding on features such as the now-deprecated Blink tag.(Mozilla, n.d.) This 'war' got so out of hand that Microsoft actually placed a giant e statue in front of the entrance to Netscape's headquarters, to which Netscape responded by defacing the statute and placing their own company mascot on top of it. (Strack, 1997)

  While this is a bit comical, all things considered, the real effect was much worse. Web developers had to support both browsers, each of which had competing features. Netscape provided users with the ability to use javascript, one of the most important programming languages in web development today, and allowing us to create responsive websites. Windows, however, provided us with cascading style sheets to make our websites look much nicer. At the time you could only use one or the other. This, along with the every increasing amount of functions that web developers had to support and the emergence of still even more browsers, it became apparent that something had to be done to standardize the web.(Mozilla, n.d.)

Web Standards & W3C

  During the Browser Wars in 1994, Tim Berners-Lee left CERN and joined with MIT to found the World Wide Web Consortium (W3C). (Berners-Lee, 2022) The mission of W3C is to create a set of standards known as Web Standards. Web Standards are the guidelines that govern the design and development of websites. These standards are driven by the need to ensure that websites are accessible to everyone regardless of device, operating system, or browser. Without these standards it is up to the browser developers to create standards for the web browser which makes it much more difficult for web developers to create websites that work in all browsers. Simplicity, interoperability, and accessibility are the key principles that guide the development of web standards. (W3C Mission)

Web Design Concepts

  Websites have a few design principles which assist developers in creating sites that comply with the W3C standards. Doing this ensures that everyone will be able to access your website, regardless of browser or device.

  • Graceful Degradation
  • Progressive/Adaptive
  • Responsive
  • Future-Friendly
  • Mobile First
Latest Trends

  The latest trend in web-development is towards the development of mobile-first websites. From Progressive Web Apps to Accelerated Mobile Pages, the focus is on mobile devices for the foreseeable future of the web. In the early days of computing, all the work was done on the server, but as the technology continues to evolve, we will likely see more and more of the work offloaded onto the client's browser.(GMI, n.d.)

  For me, personally, I'd like to see more integration between the web and generic programming languages such as Python and C. I won't have to wait long though, thanks to HTML5 we now have WebAssembly which can be used to do things like embed games written in C# into a website. This integration is still a far cry from being able to interact with the user's computer as if the code was native, but we are taking steps in the right direction.(Unity3D, n.d.)
