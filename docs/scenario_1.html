Presently

  An organization developing a site similar to presently will almost certainly find themselves having to design a website for responsiveness and accessibility. The major issue they would have in designing a responsive website would be in ensuring the page looks nice on all screen sizes. This can be accomplished by the use of CSS Media Queries to alter the stylesheet based on the width or height of the screen. It will be important to design for the smallest screen you want to support, and this can easily be done by using your browser development tools such as responsive design mode to try out your site with different screen sizes. Designing for every possible screen size is a bit tedious, instead it’s better use the min-width or max-width attributes to set the CSS for a range of screen sizes that a given styling works well on. Accessibility is another important aspect that they would want probably to focus on to ensure that everyone can use their site. One way they should do this is through the use of semantic tags such as the <article> and <section> tags. Using web accessibility checkers along with semantic validators from W3C are a great way to identify problems they may come across.

  One of the hardest parts of designing a website is laying out the site. We often know generally what pages we want and might need, but without a solid blueprint to follow it is difficult to anticipate what we might need to code to accomplish our task. Using wireframing is a great way to help developers on a team know what to code, and it also helps to organize ones’ thoughts on the general look and feel of the website. Similarly, a general outline or content map of the site is also helpful in helping us to plan out what pages we are going to need, what copywriting we need to do, and what we should make a priority. The content map tells us what content the pages will have while the wireframes tell us how that content should be displayed. Another great idea, which I did not do for my site, is to make multiple wireframes for both the desktop and mobile browsing experience. Doing this will make it easier to develop a responsive website that works well in both desktop and mobile devices.
